PHP FTP MANAGER - Desc.

The Package is intended/built to be used as a wrapper for accessing and manipulating data on FTP Servers. It extends the PHP FTP class and combines all the functions for connecting to and manipulating files and directories such as ftp_mkdir(), ftp_connect, ftp_login, ftp_get and so on into a single class.

PHP FTP Manager - Doc.

The Whole Package is broken down into three different classes stored in separate files. The first class is the Connection class stored in the ftp-connect.php file, the second is the ftp_directory class stored in the ftp-directory.php file and the last class is the ftp class which is stored in the ftp.php file.
The config data such as the server to connect to, the username and password for login and so on are stored in the parameters.ini file.

PHP FTP Manager - Conn.

The connection class is an abstract class (meaning it can't be instantiated, it can only be inherited). It creates a connection to the ftp server and logs into the users ftp account using the connection details gotten from the parameters.ini file

It contains the following Methods

  • print_error(error message to display)
  • isUsable(variable to check for)
  • parse_config_data()
  • _login()

PHP FTP Manager - FTP-Dir.

The ftp_directory class extends the connection abstract class. It contains functions for manipulating directories and working around the file structure of the ftp server

It contains the following Methods

  • manipulate_directory(action to perform, options)
  • pwd() //print current working directory
  • chdir($param), mkdir($param), rmdir($param)
  • nlist($location), rawlist($location)
"name", "goto", "display", "create" & "delete" are the only actions that can be used with the manipulate_directory method

PHP FTP Manager - FTP

The ftp class is a robust class that extends the other two classes. It also includes other methods for downloading files from the ftp server, uploading files to the server, sending SITE commands to the ftp server for commands not provided by the classes and handling errors better than the print_error(...) method in the connection class

It contains the following Methods

  • download(file to download, where to save downloaded file, file mode[Ascii or Binary])
